# Handwritten Digit Reconstruction Using RBM

This project implements a **Restricted Boltzmann Machine (RBM)** from scratch using **PyTorch** to learn and reconstruct handwritten digits from the **MNIST dataset**.

Unlike typical classification models, this project focuses on **generative modeling**, where the RBM learns the distribution of digit images and can:

* Reconstruct noisy digits
* Generate new digit samples
* Learn digit-specific representations

---

## 📌 Project Overview

* Implements RBM without using high-level deep learning abstractions
* Trains **separate RBMs for each digit (0–9)**
* Uses **Contrastive Divergence (CD-1)** for training
* Visualizes:

  * Training loss curves
  * Generated digits
  * Reconstruction results

---

## 🧠 What is an RBM?

A **Restricted Boltzmann Machine** is a two-layer neural network consisting of:

* **Visible layer** → input pixels (28×28 = 784)
* **Hidden layer** → latent features

It learns by modeling probability distributions and uses **Gibbs sampling** for reconstruction and generation.

## 🚀 How to Run

### 🔹 Train the Model

```bash
python rbm_train.py
```

This will:

* Train 10 RBMs (one per digit)
* Save the trained model as `rbm_model.pth`
* Generate training plots

---

### 🔹 Generate & Reconstruct Digits

```bash
python rbm_generate.py
```

This launches an **interactive GUI** where you can:

* Generate new handwritten digits
* Add noise and reconstruct them
* Visualize RBM outputs

## 🧪 Technical Details

| Component       | Value                         |
| --------------- | ----------------------------- |
| Visible Units   | 784 (28×28 images)            |
| Hidden Units    | 256                           |
| Training Method | Contrastive Divergence (CD-1) |
| Framework       | PyTorch                       |
